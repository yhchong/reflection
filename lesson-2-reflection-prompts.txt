What happens when you initialize a repository? Why do you need to do it?
You start a folder capable of keeping track of the changes you have made.

How is the staging area different from the working directory and the repository? What value do you think it offers?
The repository contains files that are currently being tracked in the working directory. The staging area holds the new versions of these files before they are being committed into the repository. It offers users better control over their files.

How can you use the staging area to make sure you have one commit per logical change?
By regularly committing the staging area with each logical change before proceeding to make further changes.
